A new logbook?

There is already one with a long and successful tradition: efa. This is "electronic", a single-user application. So efa still makes sense. But if you want to work digitally networked with the logbook, you will experience synchronisation and configuration problems even with the efaCloud extension. efa is simply not built for use in multiple locations. I know what I'm talking about, I also built efaCloud and have been using it in one of the largest rowing clubs in Germany since January 2021.

That's why I decided to build a new logbook that is built from the ground up for storage in the cloud and therefore supports smooth and error-free operation in multiple locations and management from home. It is based on efaCloud and is data-compatible with efa, making it easy to migrate - even back from dilbo to efa.

dilbo offers

  • Centralised management of all logbooks and inventory data (people, boats, destinations, etc.) of a club via PC and smartphone
  • Registration of trips, damage, reservations and messages messages via web browser at any location
  • The display of voyages, damage, reservations and statistics statistics via web browser at any location
  • Flexible configuration of the user interface, club data and user authorisations user authorisations.
  • Import efa backup files for migration

dilbo is growing

The 0.x versions are incomplete preliminary versions to get an impression for you and feedback for me. What is still missing is:

  • The entry of journeys, damage, reservations and messages messages with a Java programme
  • The display of journeys, damages, reservations and statistics statistics with a Java programme
  • Authorisation management for member entries: Sessions, reservations, messages.
  • Export of efa backup files for remigration
